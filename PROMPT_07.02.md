Ты — ведущий fullstack-разработчик. Работаем в проекте TripRoute (Next.js App Router + TypeScript + Tailwind + shadcn/ui + Prisma + Neon/Postgres + Auth).

ЗАДАЧА
Сделай главную страницу TripRoute: два раздела со списком публичных маршрутов:
1) Новые — сортировка по createdAt desc
2) Популярные — сортировка по количеству лайков desc
На главной показываем только PUBLIC маршруты. Без авторизации нельзя ничего править (создавать/редактировать/удалять/переключать private/public). Гость может только смотреть и (если по требованиям проекта) лайкать — но редактирование/CRUD строго запрещены.

ТРЕБОВАНИЯ К UI
- Верхнее меню (Header):
  - Лого/название TripRoute (ссылка на “/”)
  - Навигация: Главная, Каталог, Мои маршруты
  - Справа: кнопка Войти/Выйти (в зависимости от сессии), мини-аватар, имя (если есть)
- Нижнее меню (Footer):
  - Короткий текст: (с) TripRoute + год
  - Ссылки: Политика, Контакты
- Главная страница:
  - Hero-блок: заголовок + подзаголовок + кнопка “Добавить маршрут”
    - Если НЕ авторизован: кнопка ведёт на /auth/signin (или ваш путь логина) и показывается подсказка “Войдите, чтобы добавлять”
  - Два блока списков:
    - “Новые” (последние 10–20)
    - “Популярные” (топ 10–20 по likesCount)
  - Карточка маршрута в списке:
    - Заголовок, автор (имя), дата
    - Теги (если есть)
    - Кнопка “Открыть” (на страницу маршрута /routes/[id])
    - Счётчик лайков + кнопка лайка (если лайки уже реализованы)
    - НИКАКИХ кнопок “Редактировать/Удалить” на главной
- Адаптивность: красиво на мобильном.

ТРЕБОВАНИЯ К ДАННЫМ
- Сделай 2 запроса к БД (или один с двумя выборками) через Prisma:
  - recentroutes: where { isPublic: true } orderBy { createdAt: "desc" } take N
  - popularroutes: where { isPublic: true } orderBy по likesCount desc take N
- Для списка выводи likesCount через Prisma _count (или заранее посчитанное поле).
- Если пользователь авторизован — для каждой карточки добавь likedByMe (нужно для состояния кнопки лайка).
  - Реализуй это эффективно: один запрос по routeIds к таблице Like, затем мапа Set(routeId).

ПРАВА ДОСТУПА

- На главной и в публичном каталоге никакого редактирования.
- Если на проекте есть страницы редактирования — добавь guard: если нет session > redirect на login или 403.
